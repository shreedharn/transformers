    Transformer Attention, MLPs, Hidden States, RoPE, and Unembedding¶

    Big picture¶

    A transformer block mainly does two different jobs:

    1. Attention mixes information across tokens.
    2. MLP computes new features within each token position after that context has been gathered.

    A useful slogan:

    • Attention = communication across positions
    • MLP = computation within a position

    1) Token embeddings vs hidden states¶

    Token embedding¶

    The embedding layer maps each token ID to an initial vector:

    \[ x_i^{(0)} \in \mathbb{R}^{d_{model}} \]

    This is the model’s starting representation for a token.

    Examples of what an embedding loosely captures: - token identity - rough semantic similarity - lexical statistics

    Hidden state¶

    After attention and MLP layers start transforming the representation, the vector is no longer just a plain embedding. It becomes a hidden state.

    At layer \(\ell\), token \(i\) has hidden state:

    \[ h_i^{(\ell)} \in \mathbb{R}^{d_{model}} \]

    This can encode much richer information, such as: - lexical meaning - syntax - semantic role - long-range dependencies - phrase structure - prediction-relevant features

    Important distinction¶

    The later hidden state usually has the same dimensionality as the embedding vector, but not the same interpretation.

    So the model is not forcing richer meaning to fit inside the original embedding semantics. Instead, it uses a fixed-width residual stream as an evolving workspace.

    2) What attention heads do¶

    For each token, attention decides which other tokens matter.

    Very roughly:

    \[ \text{Attention}(x_i) = \sum_j \alpha_{ij} v_j \]

    where: - \(\alpha_{ij}\) = how much token \(i\) attends to token \(j\) - \(v_j\) = value vector from token \(j\)

    So attention is a form of content-dependent message passing.

    It helps answer questions like: - Which earlier token is the subject? - Which adjective modifies this noun? - Which earlier token does a pronoun refer to? - Which delimiter or bracket matters here?

    Example: white cat¶

    Suppose attention has already run. At the cat position, the representation may now contain: - cat meaning - modified by white - noun phrase context - animal-related context

    Attention is what lets cat import information from white.

    3) What the MLP does¶

    After attention, the MLP processes each token independently.

    Typical form:

    \[ \text{MLP}(x) = W_2\,\sigma(W_1x + b_1) + b_2 \]

    This means the MLP: - does not look at other tokens directly - operates on the context-enriched vector at one position - computes new nonlinear features

    Intuition¶

    If attention tells the token where to look, the MLP decides what to compute from what was found.

    Example: white cat¶

    If attention has already enriched the cat position with information from white, then the MLP can combine features like: - “cat” - “modified by adjective” - “adjective is white”

    into richer features such as: - “white-cat concept” - “animal with color attribute” - stronger noun-phrase structure features

    So: - Attention retrieves / mixes context - MLP synthesizes / transforms features

    4) Why MLP is needed if attention already transformed the vector¶

    Attention is great at: - selecting - routing - copying - aggregating information from other tokens

    But attention mainly forms weighted combinations of value vectors. The MLP adds something different: - nonlinear computation - feature synthesis - richer per-token transformations

    A good mental model: - Attention moves information - MLP processes information

    5) Dot products in transformers¶

    The dot product of two vectors \(a\) and \(b\):

    \[ a \cdot b = \sum_k a_k b_k \]

    Geometric meaning¶

    It measures alignment: - large positive = similar direction - near zero = weakly related / orthogonal - negative = opposite direction

    In attention¶

    The key attention score is:

    \[ q_i \cdot k_j \]

    Interpretation: - \(q_i\): what token \(i\) is looking for - \(k_j\): what token \(j\) offers

    So the dot product acts like a compatibility score.

    Example: white cat¶

    At the cat position, the query may strongly match the white key if the model has learned that adjective-noun relations matter.

    Then cat attends to white.

    6) Vector sums and additions in transformers¶

    Vector addition is used heavily in transformers.

    A) Weighted sum in attention¶

    After attention weights are computed, the output is a weighted sum of values:

    \[ \sum_j \alpha_{ij} v_j \]

    This combines information from relevant tokens.

    B) Residual connections¶

    Transformers usually update the hidden state like this:

    \[ x \leftarrow x + \text{Attention}(x) \]
    \[ x \leftarrow x + \text{MLP}(x) \]

    This means: - keep the current representation - add a learned update

    Example: white cat¶

    After attention, the cat representation contains white-related information. The MLP then computes another update from that enriched representation. Because of the residual connection, the context-enriched representation is carried forward, not discarded.

    So the white cat information is not typically lost after the MLP. It is preserved and refined.

    7) Sequence length vs context window¶

    Sequence length¶

    How many tokens are in the actual current input.

    Context window¶

    The maximum number of tokens the model can handle.

    So:

    \[ \text{sequence length} \le \text{context window length} \]

    Example: - model supports 8192 tokens max - current prompt has 500 tokens - sequence length = 500 - context window = 8192

    8) How many hidden states are there in a layer?¶

    If the sequence length is \(T\), then a layer has \(T\) hidden states: one per token position.

    The whole layer can be viewed as:

    \[ H^{(\ell)} \in \mathbb{R}^{T \times d_{model}} \]

    where: - one row = one token’s hidden state - one layer = all token representations at that depth

    9) Multiple attention heads vs multiple layers¶

    Multiple attention heads¶

    Heads are parallel attention mechanisms inside one layer.

    Each head: - sees the same input hidden states - learns different attention patterns - extracts a different type of relationship

    Examples of possible head behavior: - adjective–noun relations - subject–verb agreement - punctuation boundaries - coreference

    Heads are parallel specialists¶

    They work at the same depth and then their outputs are combined.

    Multiple layers¶

    Layers are sequential stages of processing.

    Layer 1 produces hidden states. Layer 2 transforms those hidden states further. Layer 3 does more, and so on.

    A rough intuition: - earlier layers: local and syntactic patterns - middle layers: phrase-level and relational patterns - later layers: more abstract and prediction-relevant features

    Compact distinction¶

    • Heads = parallel views within one layer
    • Layers = depth over multiple stages

    10) Is the representation always the same dimensionality?¶

    Usually yes in the residual stream.

    If the model dimension is \(d_{model}\), then the token representation passed from layer to layer stays in:

    \[ \mathbb{R}^{d_{model}} \]

    This is necessary because of residual additions.

    But inside the block, temporary dimensions can differ¶

    Attention heads¶

    Each head often uses a smaller internal dimension:

    \[ d_{model} = h \cdot d_{head} \]

    where: - \(h\) = number of heads - \(d_{head}\) = dimension per head

    MLP¶

    The MLP often expands and contracts the representation:

    \[ \mathbb{R}^{d_{model}} \to \mathbb{R}^{d_{ff}} \to \mathbb{R}^{d_{model}} \]

    where \(d_{ff}\) is usually larger.

    So: - residual stream dimension stays fixed - internal working dimensions can change temporarily

    11) RoPE: Rotary Positional Embedding¶

    RoPE is a positional encoding method that injects position information into attention.

    Why it is needed¶

    Attention by itself does not know token order. RoPE makes attention sensitive to position and relative distance.

    What it does¶

    Instead of adding a separate position vector to the input embedding, RoPE rotates parts of the query and key vectors by position-dependent angles.

    When it happens¶

    RoPE is applied: - after Q and K are computed - before the attention dot products are computed

    So the flow is: 1. hidden states \(x_i\) 2. compute \(q_i, k_i, v_i\) 3. apply RoPE to \(q_i\) and \(k_i\) 4. compute attention scores using the rotated Q and K

    Usually RoPE is not applied to V.

    12) How Q, K, and V are computed¶

    At a given layer, each token has hidden state:

    \[ x_i \in \mathbb{R}^{d_{model}} \]

    Then the model applies three learned linear projections:

    [ q_i = W_Q x_i + b_Q ] [ k_i = W_K x_i + b_K ] [ v_i = W_V x_i + b_V ]

    These matrices are learned during training.

    Intuition¶

    The same hidden state is transformed into three different roles: - Q: what am I looking for? - K: what do I contain? - V: what content should be passed along if attended to?

    Important point¶

    The model is not learning fixed queries or keys per token string. It is learning projection rules. So the same token can produce different Q/K/V in different contexts because its hidden state changes with context.

    13) What are \(b_Q\), \(b_K\), \(b_V\)?¶

    These are bias vectors.

    Example:

    \[ q_i = W_Q x_i + b_Q \]

    The bias shifts the output by a learned offset after the matrix multiplication.

    Conceptually: - the weight matrix mixes input features - the bias adds a baseline offset per output dimension

    Some transformer implementations include these biases; some omit them.

    14) How can richer features fit in the “same embedding space”?¶

    This was a key conceptual question.

    The answer¶

    The later hidden states are not constrained to preserve the original embedding interpretation. They only preserve the same dimensionality.

    So the later vector is not “a token embedding plus extra info.” It is an evolving hidden state in a shared fixed-width space.

    Better mental model¶

    Do not think: - the original embedding space stays semantically fixed and richer features get squeezed into it

    Instead think: - the model uses a fixed-width hidden workspace - each layer rewrites the representation into something more useful - later layers can read and transform that representation because they were trained to do so

    Why this is possible¶

    Because neural representations are distributed. Many overlapping features can be encoded in patterns across dimensions.

    15) How unembedding works if the final hidden state is very different¶

    At the end of the transformer, the model has a final hidden state:

    \[ h \in \mathbb{R}^{d_{model}} \]

    This is a contextual, prediction-oriented representation.

    The unembedding maps it to logits over the vocabulary:

    \[ \text{logits} = W_U h + b \]

    where: - \(W_U \in \mathbb{R}^{|V| \times d_{model}}\) - \(|V|\) = vocabulary size

    Key idea¶

    The unembedding is not asking:

    Does the final hidden state look like the original embedding of a token?

    It is asking:

    How much evidence does this hidden state provide for each possible next token?

    Each row of \(W_U\) is like a detector direction for one vocabulary token.

    For token \(t\):

    \[ \text{logit}_t = w_t^\top h + b_t \]

    So if the hidden state contains the right features, the corresponding output token gets a high score.

    Example¶

    If the context is: - The white cat drank some

    then the final hidden state might encode features such as: - noun likely next - liquid likely - semantically plausible continuation

    Then the unembedding may score: - milk high - water somewhat high - keyboard low

    So unembedding is a learned readout from hidden-state space to token scores.

    16) End-to-end miniature walkthrough: white cat¶

    Step 1: Embedding¶

    white and cat start as token embeddings.

    Step 2: Attention¶

    At the cat position, the query may match the white key. That gives a useful attention weight. The attention output becomes a weighted sum of values, so the cat position receives modifier information from white.

    Step 3: Residual addition¶

    The attention update is added to the existing cat representation, so the token now reflects something like: - cat - modified by white - noun phrase context

    Step 4: MLP¶

    The MLP reads that enriched representation and computes new nonlinear features, such as: - “white-cat concept” - stronger entity or phrase-level structure

    Step 5: Residual addition again¶

    The MLP output is added back, so the white cat information is preserved and refined.

    Step 6: Later layers¶

    Further layers may use that enriched state to reason about syntax, semantics, or next-token prediction.

    17) Final summary¶

    Attention heads¶

    • Mix information across tokens
    • Use dot products for relevance
    • Use weighted sums to gather context

    MLPs¶

    • Process each token independently after attention
    • Compute nonlinear feature transformations
    • Build richer abstractions from the context-enriched state

    Dot products¶

    • Measure alignment / compatibility
    • In attention: query–key matching

    Vector additions¶

    • Combine updates with existing state
    • Enable residual learning
    • Preserve and refine earlier information

    Hidden states¶

    • One per token per layer
    • Same dimensionality as the model width
    • Much richer than raw embeddings

    Heads vs layers¶

    • Heads: parallel specialists inside one layer
    • Layers: sequential stages of refinement

    RoPE¶

    • Injects positional information into attention
    • Applied to Q and K after projection, before attention scores

    Unembedding¶

    • Reads final hidden states into token logits
    • Does not require the final hidden state to resemble the original token embedding

    One final mental model¶

    A transformer uses a fixed-width residual stream as an evolving workspace.

    • Embeddings initialize the workspace
    • Attention moves information across positions
    • MLPs compute richer features within each position
    • Residual addition carries information forward while refining it
    • Unembedding reads the final hidden state and converts it into next-token scores

    So the model is not preserving a static embedding meaning from start to finish. It is repeatedly rewriting a same-width hidden representation into something more contextually useful.
